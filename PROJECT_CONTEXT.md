# Aurora — Project Context

## Overview

Aurora is a premium AI adoption platform designed to help non-technical beginners, individuals, small businesses, and lean teams use AI more effectively. The product is positioned as a practical system for choosing AI tools, learning how to use them, improving prompt quality, understanding token and cost concepts, and building workable AI-driven processes.

Aurora is not intended to be a generic prompt store or a hype-driven AI brand. It is intended to feel like a product-led SaaS platform with strong educational value, practical implementation support, and a clean upgrade path from free resources to paid subscriptions and team features.

## Core Problem

Many people want to use AI but feel overwhelmed by:
- too many AI tools
- confusing feature overlap
- unclear pricing and token costs
- weak prompting habits
- poor implementation structure
- no clear starting point
- scattered subscriptions and workflows

Aurora solves this by translating AI into a clearer, more organized, more approachable system.

## Product Vision

Aurora should become a trusted platform where users can:
- discover which AI tools are right for them
- understand what those tools do well
- learn how to prompt more effectively
- avoid overspending on AI usage
- access curated workflows and playbooks
- organize purchased resources in a dashboard
- ask an AI assistant for context and guidance
- eventually collaborate in team settings

## Product Positioning

Aurora should be positioned as:
- practical, not hypey
- premium, not flashy for its own sake
- structured, not overwhelming
- clear for beginners, but credible for professionals
- modern and AI-native, but not technical in tone

## Target Users

### Primary
- non-technical beginners
- solopreneurs
- freelancers
- creators
- consultants
- small business owners
- lean teams

### Secondary
- student builders and productivity-focused individuals
- service businesses exploring AI adoption
- teams that need curated onboarding and AI resources

## Business Model

Aurora is planned as a freemium + subscription + digital product platform.

### Free layer
- selected free resources
- starter tool guides
- limited calculators or quizzes
- limited chatbot usage later
- lead magnet / email capture

### Paid layers
- one-time digital products
- subscription access to full resource library
- premium implementation resources
- team plans with shared access later

### Possible monetization structure
- Free
- Solo
- Pro / Premium resource tier
- Team
- optional consulting or setup offers later

## UX Goals

Aurora should make users feel:
- this looks premium
- this seems trustworthy
- this is advanced but understandable
- I know where to start
- this can actually help me

Aurora should reduce intimidation at every step.

## Brand Direction

### Public site
- mostly dark premium AI SaaS aesthetic
- purple-led identity
- atmospheric aurora lighting
- bold typography
- rounded glassy surfaces
- cinematic hero motion
- polished enterprise tone

### Dashboard
- same design language, different intensity
- softer graphite / dark-gray UI
- calmer productivity tone
- reduced glow
- stronger readability and structure

## Copy Tone

Use polished enterprise copy that is:
- concise
- confident
- practical
- intelligent
- calm
- readable for non-technical users

Avoid:
- slang
- hype-heavy promises
- jargon overload
- “AI bro” tone
- academic stiffness

## Tech Stack

Planned stack:
- Next.js for the unified website + dashboard app
- Vercel for hosting/deployment
- Convex for backend, data, and application logic
- Stripe for payments, subscriptions, and billing

## Product Structure

Aurora should be built as one product, not separate disconnected properties.

Public-facing pages and logged-in dashboard should live in the same application so that:
- the brand experience stays consistent
- entitlements are easier to manage
- users do not feel bounced between systems
- purchases can unlock content inside one shared experience

## Current Build Approach

Use a front-end-first phased process:
1. build marketing site shell with mock data
2. build dashboard shell with mock data
3. add auth and onboarding
4. add product access and entitlements
5. add Stripe billing
6. add smart tools and assistant
7. add polish, analytics, and personalization

## Current Priorities

Immediate priority:
- build Sprint A marketing site in Antigravity using the visual direction brief, homepage wireframe brief, and Sprint A build prompt

Secondary priority:
- build Sprint B dashboard shell after the public-facing brand feels locked

## Non-Goals Right Now

Do not prioritize yet:
- real backend implementation
- real payments
- full team collaboration logic
- advanced personalization engine
- complete chatbot functionality
- analytics infrastructure
- production SEO strategy

## Success Criteria for Current Phase

The current phase is successful when:
- the Aurora brand feels visually locked
- the homepage is conversion-oriented and premium
- the site feels distinct from generic SaaS templates
- the product is clearly understandable from the marketing site
- the component system is reusable
- the dashboard can be built from the same design language
