# Aurora — Roadmap

## Roadmap Summary

Aurora should be developed in a staged way that prioritizes brand clarity, front-end UX quality, and a believable product experience before deeper technical systems are added.

The roadmap is intentionally front-end-first.

## Phase 0 — Planning and Memory Setup
Focus:
- product strategy
- brand direction
- phased build model
- repo memory system
- planning docs

Outcome:
- Aurora has a coherent identity and documented project context

## Phase 1 — Marketing Site v1
Focus:
- homepage
- pricing page
- library page
- product detail page
- auth entry pages
- public-site design system

Outcome:
- polished public-facing site that establishes the brand and offer

## Phase 2 — Dashboard Shell v1
Focus:
- dashboard home
- my library
- explore
- product details inside app
- onboarding flow
- chat UI shell
- billing/settings shell

Outcome:
- believable product experience with calmer dashboard UX

## Phase 3 — Auth + User State
Focus:
- signup/login
- persistent user state
- onboarding persistence
- basic personalization

Outcome:
- Aurora becomes user-aware instead of static

## Phase 4 — Access, Products, and Entitlements
Focus:
- products schema
- plans
- purchases/subscriptions mapping
- access control
- locked/unlocked content states

Outcome:
- users can have meaningful ownership/access states

## Phase 5 — Billing and Monetization
Focus:
- Stripe Checkout
- subscriptions
- one-time products
- billing portal
- payment-to-access flow

Outcome:
- Aurora becomes commercially functional

## Phase 6 — Interactive Product Value
Focus:
- AI assistant
- tool finder
- token/cost calculator
- prompt helper
- recommendation logic

Outcome:
- Aurora becomes more than a resource library

## Phase 7 — Retention and Optimization
Focus:
- personalization refinement
- saved items
- upsell paths
- analytics
- onboarding improvements
- content/recommendation improvements

Outcome:
- better retention, conversion, and product usefulness

## Phase 8 — Team Expansion
Focus:
- organizations
- seats
- admin views
- team resource sharing
- team onboarding kits

Outcome:
- Aurora becomes more viable for business/team plans

## MVP Definition

Aurora’s MVP should likely include:
- premium marketing site
- dashboard shell
- onboarding
- basic auth
- library/product pages
- access control
- Stripe billing
- initial downloadable resources

## Post-MVP Priorities

After MVP, prioritize:
- assistant usefulness
- calculators and recommendations
- stronger personalization
- team access
- content expansion

## Key Product Bets

Aurora is betting that users will pay for:
- clarity around AI tool choice
- practical implementation help
- curated resource libraries
- simplified cost/token education
- better prompts and workflow systems
- a unified place to manage purchased resources

## Risks to Watch

- overdesigning the marketing site while under-explaining the product
- building too much backend before the UX is validated
- making the AI theme too intense for beginners
- drifting into a generic prompt-store brand
- weak differentiation from other AI education products
