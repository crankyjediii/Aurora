Use the block below as the website-shell build prompt for Antigravity.

```text
Build the front-end marketing website for a premium AI SaaS brand called
Aurora.

Aurora is an AI adoption platform for non-technical beginners, individuals,
small businesses, and lean teams. It helps users choose the right AI tools,
understand what they do, learn how to use them, and build smarter workflows
with less confusion.

This is not a generic prompt marketplace. It is a premium product-led SaaS
platform focused on practical AI adoption.

Core visual direction:
Design Aurora as a mostly dark premium SaaS website with:
- a purple-led brand identity
- atmospheric aurora-like lighting
- subtle indigo / blue support accents
- bold typography
- rounded premium UI
- soft glassmorphism where appropriate
- strong spacing
- layered depth
- tasteful motion
- a high-impact animated hero

The website should feel:
- premium
- intelligent
- trustworthy
- approachable
- cutting-edge
- polished
- futuristic without being intimidating

Avoid making it feel:
- crypto / web3
- cyberpunk
- cartoonish
- generic startup template
- boring enterprise SaaS
- neon overload
- cluttered

Audience:
- non-technical beginners
- solopreneurs
- freelancers
- creators
- consultants
- small business owners
- light teams

Copy tone:
Use polished enterprise copy:
- concise
- confident
- practical
- calm
- clear
- not hypey
- not slangy
- not overly technical

Important style rule:
The public site should be more dramatic and cinematic than the future
dashboard. This website should hook attention and sell the product, while still
feeling clean and trustworthy.

Build scope:
Create these public-facing pages using mock data only:
1. Homepage
2. Pricing page
3. Product library / resources page
4. Product detail page
5. Login page
6. Signup page
7. Shared responsive navigation
8. Shared responsive footer

Technical requirements:
- use Next.js
- keep the code organized for future dashboard expansion in the same app
- use reusable components
- make it fully responsive
- prioritize desktop-first polish but make mobile solid
- use mock data only for now
- no backend, no Stripe, no Convex yet

Design system requirements:
Create reusable components for:
- navbar
- buttons
- section headers
- feature cards
- product cards
- pricing cards
- FAQ accordion
- footer
- form inputs
- badges
- dashboard preview panels
- CTA banners

Homepage requirements:
Build the homepage with these sections in order:

1. Top navigation
- Aurora logo
- Solutions
- Library
- Pricing
- Learn
- Login
- Start Free CTA
Use a premium dark translucent or matte nav with subtle blur and rounded
treatment.

2. Hero section
This is the most important section.
Include:
- eyebrow label
- large bold headline
- short subheadline
- primary CTA: Start Free
- secondary CTA: Explore the Library or See How It Works
- compact value / trust strip
- large right-side visual combining abstract aurora glow and realistic product
  UI mockups
Use layered floating cards and premium motion.
Hero motion should include:
- slow aurora drift
- subtle floating UI panels
- soft parallax
- tasteful glow animation
Make it feel cinematic, premium, and conversion-focused.

3. Trust / value strip
Show a short horizontal strip with value areas like:
- Tool Guides
- Prompt Systems
- Cost Optimization
- Workflow Kits
- Team Resources

4. Problem / pain section
Show the user pain clearly:
- too many AI tools
- unclear pricing
- weak prompts
- fragmented workflows
- no clear starting point
Use a premium grid of pain-point cards.

5. Solution section
Explain Aurora as one practical system for AI adoption:
- choose the right tools
- access curated guidance
- build smarter workflows
Use clean feature blocks.

6. Product category section
Create category cards for:
- Tool Guides
- Prompt Systems
- Cost & Token Resources
- Workflow Kits
- Team Implementation Packs

7. Dashboard preview section
Show realistic previews of the app in a calmer, more structured visual style
than the hero.
This should visually bridge the marketing site and the product.

8. How it works section
Use a simple 3-step explanation:
1. Find your fit
2. Unlock practical resources
3. Build smarter workflows

9. Pricing teaser
Show three plans:
- Free
- Solo
- Team
Use elegant pricing cards and highlight one plan subtly.

10. FAQ
Use a clean dark accordion with readable spacing.

11. Final CTA
Create a refined closing section with premium glow accents and strong
call-to-action buttons.

12. Footer
Make it minimal, premium, and structured.

Pricing page requirements:
- strong heading
- concise explanation
- Free / Solo / Team pricing cards
- clear feature summaries
- recommended plan emphasis
- FAQ section
- CTA section

Library / resources page requirements:
- curated product catalog feeling
- search input UI
- category filters
- featured resources
- premium product cards
- clean metadata
- dark premium layout

Product detail page requirements:
- product title
- category
- short positioning statement
- what's included
- who it's for
- preview section
- CTA section
- related resources
- premium dark layout

Login / signup page requirements:
- centered dark premium auth card
- strong typography
- simple form fields
- polished inputs
- consistent Aurora styling
- mock-only flow

Motion / interaction requirements:
Use tasteful motion across the site:
- smooth section reveals
- hover lift on cards
- glow transitions
- button polish
- subtle layered motion in hero visuals

Do not over-animate the entire site. Keep the strongest visual motion in the
hero and use restrained polish elsewhere.

End goal for Sprint A:
Deliver a polished, production-quality front-end website that locks in Aurora's
brand identity and public user experience before any backend or billing logic
is added.
```
