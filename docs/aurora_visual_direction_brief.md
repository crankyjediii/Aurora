# Brand summary

**Aurora** is a premium AI adoption platform for non-technical beginners,
individuals, small businesses, and lean teams. It helps people choose the right
AI tools, understand what they do, use them more effectively, and build better
workflows without feeling overwhelmed.

Aurora should look advanced, but never intimidating. It should feel like a
high-end modern SaaS product with a dark premium aesthetic, purple-led
lighting, polished enterprise structure, and a calmer productivity-focused
dashboard experience.

The core brand idea is:

**Cutting-edge AI, made clear and usable.**

# Brand personality

Aurora should feel:

- premium
- intelligent
- trustworthy
- approachable
- modern
- polished
- structured
- calm
- future-facing

Aurora should not feel:

- gimmicky
- crypto-like
- overly flashy
- chaotic
- cartoonish
- too technical
- too corporate and lifeless
- trendy in a cheap way

# Design keywords

Use these words as the design north star:

- luminous
- premium
- dark
- atmospheric
- elegant
- structured
- modern
- intelligent
- clear
- trustworthy
- beginner-safe
- cinematic
- refined

# Visual style

Aurora's style should be:

**Mostly dark premium AI SaaS with purple aurora glow, bold typography, glassy
rounded surfaces, strong hierarchy, and tasteful motion.**

The public site should be more dramatic and visually magnetic. The dashboard
should use the same design language, but with lower intensity: softer gray-dark
backgrounds, less glow, more utility, and more readability.

That means:

**Same brand language, different intensity.**

# Color palette direction

Aurora should be clearly purple-led.

## Core palette mood

- near-black backgrounds
- deep charcoal and graphite surfaces
- rich purple as primary accent
- indigo / cool blue as secondary light support
- off-white primary text
- cool grays for secondary text and UI labels

## Public site color behavior

- darker backgrounds
- stronger purple/indigo glow
- more contrast
- atmospheric gradients
- subtle bloom effects around hero visuals

## Dashboard color behavior

- softer dark gray surfaces
- less black void
- muted violet accent for active states
- controlled use of glow
- clearer panels and section separation

## Palette guidance

The purple should feel premium and atmospheric, not neon nightclub. The blue
should support the purple, not overpower it. Glow should feel like light in
air, not LED strips.

Avoid:

- bright hot pink as a dominant color
- aggressive neon
- oversaturated rainbow gradients
- generic enterprise blue-only palettes
- green "matrix AI" looks

# Typography direction

Typography should carry a lot of the premium feel.

## Style

Use a clean modern sans serif that feels:

- precise
- high-end
- highly readable
- slightly sharp, not playful
- strong in large hero sizes
- suitable for polished enterprise copy

## Headline behavior

- large
- bold
- concise
- calm and declarative
- not too wordy
- strong contrast against the background

## Body copy behavior

- easy to scan
- not too dense
- plain-English
- confident and useful
- never jargon-heavy unless clearly explained

## UI text behavior

- crisp and readable
- consistent hierarchy
- strong clarity
- comfortable spacing

# Spacing and corner radius style

Aurora should feel spacious and expensive.

## Spacing

- generous section spacing
- large breathing room around hero content
- comfortable interior card padding
- strong whitespace between major blocks
- avoid cramped layouts

## Corner style

- rounded, but not bubbly
- premium large-radius cards
- pill buttons for primary actions
- soft rounded panels and containers
- the shape language should feel controlled and elegant

# Button, card, and input style

## Buttons

### Primary buttons

- purple-led
- high contrast
- slightly luminous
- premium hover state
- should feel like a polished CTA, not a generic utility button

### Secondary buttons

- darker or outlined
- subtle border
- restrained but still premium
- should pair well beside the primary hero action

## Cards

Cards should feel layered and premium, not flat.

Shared qualities:

- rounded corners
- soft border or low-contrast outline
- subtle shadow/depth
- optional glass or matte translucent treatment depending on section
- hover lift or glow polish
- calm spacing and alignment

Use different card roles:

- feature cards
- pain-point cards
- product cards
- pricing cards
- resource cards
- dashboard preview cards

## Inputs

Inputs should be:

- dark but readable
- slightly elevated
- clean and modern
- focused on clarity
- subtle focus glow or border shift
- never overly thin or hard to read

# Motion rules

Motion is important, but it should feel premium and intentional.

## Overall motion philosophy

Motion should feel:

- smooth
- atmospheric
- elegant
- slow where dramatic
- fast where functional

Motion should not feel:

- noisy
- distracting
- gimmicky
- hyperactive
- game-like

## Public website motion

Use stronger motion in the hero and slightly reduced motion elsewhere.

Hero motion can include:

- drifting aurora gradients
- layered floating UI cards
- slow parallax
- soft light movement
- subtle orbital paths or lines
- polished CTA hover reactions

Below the hero:

- fade and slide reveals
- hover lifts
- gentle card glow transitions
- accordion animations
- restrained section entry animations

## Dashboard motion

The dashboard should feel calmer:

- quick page fades
- smooth tab changes
- hover emphasis
- focus state transitions
- soft dropdown/modal movement
- no dramatic glow choreography

# Hero art direction

The hero should be visually magnetic and clearly AI/product-led.

It should combine:

- abstract aurora-like purple lighting
- realistic product UI mockups or interface fragments
- depth layers
- floating cards or panels
- subtle visual motion
- clean text on the left, strong visual on the right

It should not be:

- generic robot imagery
- overly abstract with no product relevance
- cluttered with too many floating objects
- a fake sci-fi illustration detached from the actual app

The ideal effect is:

**"This looks advanced and premium, and I can already see the product."**

# Public site vs dashboard style split

## Public site

**Purpose**

- hook attention fast
- create brand desire
- build trust
- convert the visitor into signup or exploration

**Style**

- darker
- more atmospheric
- stronger purple light treatment
- more cinematic hero
- larger statements
- more visual flare

## Dashboard

**Purpose**

- reduce friction
- make the platform usable
- help users find what they need
- feel reliable daily

**Style**

- softer graphite / gray-dark surfaces
- lower contrast
- reduced glow
- clearer structure
- more functional composition
- calmer visual rhythm

# Avoid list

Aurora should avoid:

- crypto / Web3 visual language
- cartoon illustrations as the primary style
- generic AI brains or robots
- harsh cyberpunk neon overload
- overly dense enterprise layouts
- tiny text in dark mode
- decorative clutter
- too many gradients competing at once
- "futuristic" visuals with no product clarity
- cheesy startup stock photography
- loud gamified interfaces

# Brand summary sentence

**Aurora is a premium dark-mode AI SaaS brand with purple atmospheric glow,
strong product-led visuals, and a calmer productivity-focused app experience
designed to make advanced AI feel clear and approachable.**
