# Document 1 - Aurora Visual Direction Brief

---

# Brand summary

**Aurora** is a premium AI adoption platform for non-technical beginners,
individuals, small businesses, and lean teams. It helps people choose the right
AI tools, understand what they do, use them more effectively, and build better
workflows without feeling overwhelmed.

Aurora should look advanced, but never intimidating. It should feel like a
high-end modern SaaS product with a dark premium aesthetic, purple-led
lighting, polished enterprise structure, and a calmer productivity-focused
dashboard experience.

The core brand idea is:

**Cutting-edge AI, made clear and usable.**

# Brand personality

Aurora should feel:

- premium
- intelligent
- trustworthy
- approachable
- modern
- polished
- structured
- calm
- future-facing

Aurora should not feel:

- gimmicky
- crypto-like
- overly flashy
- chaotic
- cartoonish
- too technical
- too corporate and lifeless
- trendy in a cheap way

# Design keywords

Use these words as the design north star:

- luminous
- premium
- dark
- atmospheric
- elegant
- structured
- modern
- intelligent
- clear
- trustworthy
- beginner-safe
- cinematic
- refined

# Visual style

Aurora's style should be:

**Mostly dark premium AI SaaS with purple aurora glow, bold typography, glassy
rounded surfaces, strong hierarchy, and tasteful motion.**

The public site should be more dramatic and visually magnetic. The dashboard
should use the same design language, but with lower intensity: softer gray-dark
backgrounds, less glow, more utility, and more readability.

That means:

**Same brand language, different intensity.**

# Color palette direction

Aurora should be clearly purple-led.

## Core palette mood

- near-black backgrounds
- deep charcoal and graphite surfaces
- rich purple as primary accent
- indigo / cool blue as secondary light support
- off-white primary text
- cool grays for secondary text and UI labels

## Public site color behavior

- darker backgrounds
- stronger purple/indigo glow
- more contrast
- atmospheric gradients
- subtle bloom effects around hero visuals

## Dashboard color behavior

- softer dark gray surfaces
- less black void
- muted violet accent for active states
- controlled use of glow
- clearer panels and section separation

## Palette guidance

The purple should feel premium and atmospheric, not neon nightclub. The blue
should support the purple, not overpower it. Glow should feel like light in
air, not LED strips.

Avoid:

- bright hot pink as a dominant color
- aggressive neon
- oversaturated rainbow gradients
- generic enterprise blue-only palettes
- green "matrix AI" looks

# Typography direction

Typography should carry a lot of the premium feel.

## Style

Use a clean modern sans serif that feels:

- precise
- high-end
- highly readable
- slightly sharp, not playful
- strong in large hero sizes
- suitable for polished enterprise copy

## Headline behavior

- large
- bold
- concise
- calm and declarative
- not too wordy
- strong contrast against the background

## Body copy behavior

- easy to scan
- not too dense
- plain-English
- confident and useful
- never jargon-heavy unless clearly explained

## UI text behavior

- crisp and readable
- consistent hierarchy
- strong clarity
- comfortable spacing

# Spacing and corner radius style

Aurora should feel spacious and expensive.

## Spacing

- generous section spacing
- large breathing room around hero content
- comfortable interior card padding
- strong whitespace between major blocks
- avoid cramped layouts

## Corner style

- rounded, but not bubbly
- premium large-radius cards
- pill buttons for primary actions
- soft rounded panels and containers
- the shape language should feel controlled and elegant

# Button, card, and input style

## Buttons

### Primary buttons

- purple-led
- high contrast
- slightly luminous
- premium hover state
- should feel like a polished CTA, not a generic utility button

### Secondary buttons

- darker or outlined
- subtle border
- restrained but still premium
- should pair well beside the primary hero action

## Cards

Cards should feel layered and premium, not flat.

Shared qualities:

- rounded corners
- soft border or low-contrast outline
- subtle shadow/depth
- optional glass or matte translucent treatment depending on section
- hover lift or glow polish
- calm spacing and alignment

Use different card roles:

- feature cards
- pain-point cards
- product cards
- pricing cards
- resource cards
- dashboard preview cards

## Inputs

Inputs should be:

- dark but readable
- slightly elevated
- clean and modern
- focused on clarity
- subtle focus glow or border shift
- never overly thin or hard to read

# Motion rules

Motion is important, but it should feel premium and intentional.

## Overall motion philosophy

Motion should feel:

- smooth
- atmospheric
- elegant
- slow where dramatic
- fast where functional

Motion should not feel:

- noisy
- distracting
- gimmicky
- hyperactive
- game-like

## Public website motion

Use stronger motion in the hero and slightly reduced motion elsewhere.

Hero motion can include:

- drifting aurora gradients
- layered floating UI cards
- slow parallax
- soft light movement
- subtle orbital paths or lines
- polished CTA hover reactions

Below the hero:

- fade and slide reveals
- hover lifts
- gentle card glow transitions
- accordion animations
- restrained section entry animations

## Dashboard motion

The dashboard should feel calmer:

- quick page fades
- smooth tab changes
- hover emphasis
- focus state transitions
- soft dropdown/modal movement
- no dramatic glow choreography

# Hero art direction

The hero should be visually magnetic and clearly AI/product-led.

It should combine:

- abstract aurora-like purple lighting
- realistic product UI mockups or interface fragments
- depth layers
- floating cards or panels
- subtle visual motion
- clean text on the left, strong visual on the right

It should not be:

- generic robot imagery
- overly abstract with no product relevance
- cluttered with too many floating objects
- a fake sci-fi illustration detached from the actual app

The ideal effect is:

**"This looks advanced and premium, and I can already see the product."**

# Public site vs dashboard style split

## Public site

**Purpose**

- hook attention fast
- create brand desire
- build trust
- convert the visitor into signup or exploration

**Style**

- darker
- more atmospheric
- stronger purple light treatment
- more cinematic hero
- larger statements
- more visual flare

## Dashboard

**Purpose**

- reduce friction
- make the platform usable
- help users find what they need
- feel reliable daily

**Style**

- softer graphite / gray-dark surfaces
- lower contrast
- reduced glow
- clearer structure
- more functional composition
- calmer visual rhythm

# Avoid list

Aurora should avoid:

- crypto / Web3 visual language
- cartoon illustrations as the primary style
- generic AI brains or robots
- harsh cyberpunk neon overload
- overly dense enterprise layouts
- tiny text in dark mode
- decorative clutter
- too many gradients competing at once
- "futuristic" visuals with no product clarity
- cheesy startup stock photography
- loud gamified interfaces

# Brand summary sentence

**Aurora is a premium dark-mode AI SaaS brand with purple atmospheric glow,
strong product-led visuals, and a calmer productivity-focused app experience
designed to make advanced AI feel clear and approachable.**

# Document 2 - Aurora Homepage Wireframe Brief

---

The Aurora homepage should do four things:

1. hook the user in under five seconds
2. explain what Aurora is
3. make the product feel real and useful
4. lead users toward signup, exploration, or pricing

# Section 1 - Top nav

## Goal

Orient the visitor immediately and give them a clear action path.

## Copy direction

Minimal, polished, enterprise-like.

## Layout

- left: Aurora wordmark/logo
- center or right: Solutions, Library, Pricing, Learn
- far right: Login, Start Free

## UI elements

- translucent or matte dark nav bar
- rounded or softly inset container
- subtle blur
- premium CTA button

## Motion / visual notes

- slight hover polish
- nav should feel steady and confident, not flashy

# Section 2 - Hero

## Goal

Hook attention immediately and communicate value with emotional pull.

## Headline direction

Bold, concise, premium.

Example directions:

- Adopt AI with clarity.
- Make AI practical for work and life.
- Turn AI into a system, not a guessing game.
- Modern AI guidance for people and teams.

## Supporting copy direction

Practical and polished. Explain that Aurora helps users choose the right tools,
learn what they do, and build better workflows without confusion.

## CTA

Primary: Start Free  
Secondary: Explore the Library or See How It Works

## Layout

Two-column desktop layout.

**Left side**

- eyebrow label
- headline
- short subheadline
- two CTAs
- compact trust/value strip

**Right side**

- large layered visual
- realistic dashboard/UI fragments
- abstract aurora glow
- floating cards
- depth and motion

## Content blocks

- hero eyebrow
- main headline
- subcopy
- CTA row
- mini value strip
- visual composition

## UI elements

- hero buttons
- glow panels
- floating product cards
- product screenshot fragments
- subtle badges or stats if needed

## Motion / visual notes

This is the main spectacle area.

- slow purple drift
- floating panels
- soft parallax
- controlled glow pulses
- strong polish, no chaos

# Section 3 - Trust strip

## Goal

Give the visitor quick reassurance and make the offer feel tangible.

## Copy direction

Short phrases only.

Examples:

- Tool Guides
- Prompt Systems
- Cost Optimization
- Workflow Kits
- Team Resources

## Layout

Horizontal strip beneath the hero.

## Content blocks

- 4 to 6 value labels
- subtle icon or separator system

## UI elements

- horizontal badges or text row
- low-profile styling

## Motion / visual notes

Minimal motion. Keep it clean and fast to scan.

# Section 4 - Problem / pain section

## Goal

Make the user feel understood and show that Aurora solves a real problem.

## Headline direction

Examples:

- AI adoption is still too fragmented.
- Most people don't need more tools. They need clarity.

## Supporting copy direction

Explain the real pain:

- too many AI tools
- unclear pricing
- weak prompts
- disconnected subscriptions
- no idea where to start
- too much noise and not enough guidance

## Layout

Section header followed by a card grid.

## Content blocks

- section label
- headline
- short paragraph
- 3 to 5 pain cards

## UI elements

- dark cards
- simple icons or visual accents
- concise copy

## Motion / visual notes

Subtle card hover response only. Keep this section more grounded than the hero.

# Section 5 - Solution section

## Goal

Position Aurora as a structured answer to the pain.

## Headline direction

Examples:

- One system for practical AI adoption.
- Everything you need to move from confusion to clarity.

## Supporting copy direction

Explain that Aurora helps users:

- choose tools more intelligently
- access curated learning and implementation resources
- organize AI into practical workflows

## Layout

Three-column or staggered solution blocks.

## Content blocks

- tool selection
- guided resources
- practical implementation

## UI elements

- feature cards or panels
- icon + heading + short explanation

## Motion / visual notes

Softer reveal animations. This section should feel confident and explanatory.

# Section 6 - Product category section

## Goal

Show what the platform actually includes.

## Headline direction

Examples:

- Built around practical AI use cases.
- Explore the tools, systems, and resources inside Aurora.

## Supporting copy direction

Show that Aurora is more than a vague AI platform. It has specific product
buckets users can understand quickly.

## Layout

Grid of category cards.

## Content blocks

Suggested categories:

- Tool Guides
- Prompt Systems
- Cost & Token Resources
- Workflow Kits
- Team Implementation Packs

## UI elements

- product category cards
- short descriptions
- icons or visual tags
- optional "learn more" hover state

## Motion / visual notes

Cards can lift slightly or glow subtly on hover.

# Section 7 - Dashboard preview section

## Goal

Make the platform feel real and bridge the brand site to the product
experience.

## Headline direction

Examples:

- A calmer way to work with AI.
- Everything you need, in one guided workspace.

## Supporting copy direction

Explain that the dashboard brings owned products, recommendations, and support
into one organized environment.

## Layout

Split layout or stacked layout with large product preview imagery.

## Content blocks

- dashboard home preview
- library preview
- chat/tool finder preview
- short feature captions

## UI elements

- realistic UI mockups
- floating callout tags
- highlight cards

## Motion / visual notes

Less dramatic than the hero. This should feel polished and believable.

# Section 8 - How it works

## Goal

Reduce friction by explaining the process simply.

## Headline direction

Examples:

- Get started in three clear steps.
- A simple path to better AI adoption.

## Supporting copy direction

Short, plain-English.

## Layout

Three-step row or vertical stack.

## Content blocks

1. Find your fit
2. Unlock practical resources
3. Build smarter workflows

## UI elements

- numbered cards or step panels
- compact icons

## Motion / visual notes

Minimal motion. Keep this highly readable.

# Section 9 - Pricing teaser

## Goal

Introduce plans and encourage deeper consideration.

## Headline direction

Examples:

- Start free. Upgrade when you're ready.
- Pricing designed for individuals and teams.

## Supporting copy direction

Keep it simple. Emphasize clarity and flexibility.

## Layout

Three-column pricing cards.

## Content blocks

- Free
- Solo
- Team

## UI elements

- pricing cards
- feature bullets
- one highlighted recommended plan
- CTA buttons

## Motion / visual notes

Subtle hover emphasis only.

# Section 10 - FAQ

## Goal

Remove hesitation and answer common objections.

## Copy direction

Concise, clear, trust-building.

Suggested questions:

- What is Aurora for?
- Is Aurora for beginners?
- Do I need technical experience?
- What kinds of resources are included?
- Is Aurora a subscription or one-time purchase?
- Can teams use Aurora together?

## Layout

Single-column accordion.

## Content blocks

- FAQ heading
- short intro
- accordion list

## UI elements

- dark accordion panels
- readable spacing
- subtle expand animation

## Motion / visual notes

Simple and smooth.

# Section 11 - Final CTA

## Goal

End the page with confidence and action.

## Headline direction

Examples:

- Start with clarity.
- Build your AI stack the smarter way.
- Use AI with more confidence and less guesswork.

## Supporting copy direction

Short and strong.

## CTA

Primary: Start Free  
Secondary: Explore Pricing

## Layout

Centered CTA block with atmospheric accent treatment.

## Content blocks

- headline
- short supporting copy
- CTA row

## UI elements

- premium call-to-action block
- glow edge or gradient atmosphere

## Motion / visual notes

Refined, not loud. This should feel like a polished closing invitation.

# Section 12 - Footer

## Goal

Finish with structure and trust.

## Layout

Multi-column footer.

## Content blocks

- Aurora logo and mission line
- product links
- pricing
- learn/resources
- legal
- social later if desired

## UI elements

- dark footer grid
- simple dividers
- restrained text styling

## Motion / visual notes

Very light hover effects only.
